## 開發偏好與工作流程

### 需求釐清
- 如果需求不清楚或有歧義，必須先提出問題確認
- 優先確理解正確，避免做錯方向的開發

### 任務分解
- 偏好小步驟、小改動的開發方式
- 收到需求後，協助將其拆解為可執行的小任務（tasks）
- 每個 task 應該要：
  - 儘可能小且獨立
  - 有明確的編號（如 Task 1, Task 2...）
  - 有清楚的目標和範圍
  - 可以在短時間內完成
  - 由使用者決定要先執行哪個 task
  - 一次專注於一個 task，完成後再進行下一個

### 回應格式
- 除了技術名詞之外，盡量使用繁體中文回覆

## 開發環境

- 使用 fvm 執行 flutter 或 dart 指令
